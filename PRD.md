# Learning Loop AI – PRD

## 一、产品概述

**产品名称**：Learning Loop AI  
**一句话定位**：AI驱动的个人学习系统，通过“讲解 → 出题 → 错题强化”闭环，帮助备考用户从“听懂”到“会做”。

**核心价值**  
- 用户输入任意知识点 → AI生成结构化讲解 + 一道选择题 → 用户作答 → 若错误，AI分析错因并生成相似题，形成学习闭环。

**目标**  
用最小可行产品（MVP）验证产品逻辑，证明“一人+AI”能够快速构建完整教育产品，用于个人简历投递与能力展示。

---

## 二、目标用户

- **备考人群**：CFA、CPA、FRM自学者  
- **用户特征**：时间碎片化，容易“看懂但不会做”，需要即时反馈  
- **使用场景**：自习、刷题间隙、知识查漏补缺

---

## 三、核心功能（MVP）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 知识点输入 | 文本输入框，用户自由输入问题（如“什么是NPV？”） | P0 |
| AI讲解生成 | 调用AI生成结构化内容：通俗解释、举例说明、常见误区 | P0 |
| AI题目生成 | 基于同一知识点，生成1道选择题（4个选项） | P0 |
| 答题与反馈 | 用户选择答案后，系统判断对错；若错误，展示错因分析 + 正确思路 + “再来一道”按钮 | P0 |
| 再来一道 | 基于相同知识点生成一道相似新题，重置答题状态 | P0 |

---

## 四、用户流程

```
进入页面
  ↓
输入知识点（如“什么是净现值NPV？”）
  ↓
点击“生成讲解 + 题目”
  ↓
页面展示：
  - AI讲解（三段结构）
  - 题目（问题+4个选项）
  ↓
用户选择一个选项
  ↓
系统判断：
  - 正确：显示“✅ 回答正确！”
  - 错误：显示错因分析、正确思路，并显示“再来一道”按钮
  ↓
（若错误）用户点击“再来一道”
  ↓
刷新题目（新题，同知识点），清空选中状态，回到答题环节
```

## 五、界面设计风格

**基调**：浅卡其色、高级沉浸式、现代化简约高端

### 1. 颜色系统
- 背景主色：`#F5F0E8`（浅卡其）
- 卡片底色：`#FFFFFF`（暖白）
- 强调色：`#C4A484`（暖褐）
- 文字主色：`#2C2A28`（深灰）
- 辅助文字：`#7A6E64`（中灰）
- 边框：`#E2DCD4`（浅灰）
- 成功反馈：`#6B8E23`（橄榄绿）
- 错误反馈：`#CD5C5C`（陶土红）

### 2. 字体与排版
- 英文字体：Inter
- 中文字体：PingFang SC, Microsoft YaHei
- 标题：20px/18px Semibold
- 正文：16px Regular，行高1.5
- 说明文字：14px

### 3. 布局（桌面端）
- 主内容区宽度：1000px，居中
- 各模块垂直间距：40px
- 卡片内边距：24px
- 圆角：卡片16px，按钮/输入框12px
- 阴影：极浅阴影，保持通透感

### 4. 组件风格
- 主按钮：暖褐背景，白色文字，悬停变深
- 次要按钮：透明背景，暖褐边框
- 输入框：白色背景，浅灰边框，聚焦变暖褐
- 单选选项：圆形按钮，选中时暖褐填充
- 反馈卡片：白色背景，左侧彩色边框（绿色/红色）

## 六、技术方案

| 层级 | 技术选型 | 用途 |
|------|----------|------|
| 前端框架 | Next.js (React) | 构建单页应用，处理UI与API调用 |
| 后端 | Next.js API Routes | 作为AI请求的转发层 |
| 数据库 | Supabase (PostgreSQL) | 存储学习记录（可选，MVP可先不用） |
| AI服务 | OpenAI API / 其他LLM | 生成讲解、题目、错因分析 |
| 部署 | Vercel | 一键部署，提供公网访问 |
| 版本管理 | Git + GitHub | 代码托管，展示开发过程 |

## 七、数据设计（可选，建议MVP阶段先不存储，后续可快速接入）

若后期接入Supabase，设计一张表 `learning_records`：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_input | TEXT | 用户输入的知识点 |
| ai_explain | TEXT | AI讲解内容 |
| question | TEXT | 题目文本 |
| options | JSONB | 四个选项 |
| correct_answer | CHAR(1) | 正确答案（A/B/C/D） |
| user_answer | CHAR(1) | 用户选择答案 |
| is_correct | BOOLEAN | 是否正确 |
| feedback | TEXT | 错因分析（仅错误时） |
| created_at | TIMESTAMP | 创建时间 |

该表用于后续分析错题率、二次练习率，支撑产品迭代决策。

## 八、核心指标（MVP阶段重点关注）

- **功能完成率**：用户输入 → 完成做题的比例
- **错题率**：初次答题错误的比例
- **二次练习率**：错误后是否点击“再来一道”
- **AI内容质量**：用户是否愿意继续使用（可通过Tally收集主观反馈）

## 附录：AI Prompt模板（供参考）

### 讲解生成Prompt
```
你是一位专业的考试辅导老师。请用以下结构讲解用户的问题：
1. 通俗解释
2. 举例说明
3. 常见误区

用户问题：{user_input}
```

### 题目生成Prompt
```
基于以下知识点，生成1道单项选择题，包含4个选项，并给出正确答案（A/B/C/D）。
知识点：{user_input}
输出格式：
问题：...
A. ...
B. ...
C. ...
D. ...
正确答案：X
```

### 错题强化Prompt
```
用户答错了以下题目：
题目：{question}
正确答案：{correct_answer}
用户选择：{user_answer}

请：
1. 说明为什么这个答案是错的
2. 给出正确思路
3. 生成一道类似的题目（选择题，4个选项，给出答案）
```
